## The Problem

Large language models had two largely separate superpowers by 2022: *reasoning* (chain-of-thought prompting) and *acting* (generating action plans in environments). But keeping them separate caused real problems. Chain-of-thought is a static, closed-loop process — the model reasons entirely from its internal weights, with no ability to look anything up or correct a wrong assumption mid-stream. This leads to hallucination and error propagation: once the model goes down a wrong reasoning path, nothing pulls it back.

On the acting side, systems like SayCan or WebGPT could query external tools or navigate environments, but they had no internal monologue. They couldn't maintain a high-level plan, handle unexpected situations, or explain *why* they were taking an action. They were fast but brittle — pure reactive systems with no strategic layer.

The result was a gap: no system could simultaneously think through a problem *and* reach out to the real world to verify or gather information mid-thought. Humans do this constantly — you pause cooking to search a recipe, adjust the plan when an ingredient is missing, and narrate progress to yourself along the way.

## The Idea

**Interleave thoughts and actions in the same token stream.** Instead of reasoning then acting (or acting without reasoning), ReAct generates alternating `Thought → Action → Observation` triplets in a single prompt, letting reasoning inform actions and observations update reasoning in real time.

The key intuition is dead simple: if you let the model *talk to itself* between API calls, it can track what it's learned, decide what to do next, and recover from mistakes. The "inner speech" that humans use for self-regulation is just... also useful for LLMs.

## How It Works

The setup is a prompted LLM that produces trajectories of the form:

```
Thought: I need to find what device Apple Remote was originally designed to control.
Act: Search[Apple Remote]
Obs: The Apple Remote is a remote control introduced in October 2005 by Apple, originally designed to control the Front Row media center program...
Thought: Apple Remote was originally designed for Front Row. Now I need to find what else can control Front Row.
Act: Search[Front Row software]
Obs: Front Row is a discontinued media center software... controlled by an Apple Remote or the keyboard function keys.
Act: Finish[keyboard function keys]
```

Step by step:
1. **Prompt construction**: A few-shot prompt (1–2 examples) shows the model the `Thought/Act/Obs` format. No fine-tuning needed.
2. **Action types**: Depending on the task — `Search[query]`, `Lookup[term]`, `Finish[answer]` for QA; navigation/manipulation actions for embodied tasks.
3. **External grounding**: After each `Act`, a real tool is called (Wikipedia API, web browser simulator, or text game engine) and the result is appended as `Obs`.
4. **Reasoning traces**: `Thought` steps are free-form text generated by the LLM. They serve as working memory — tracking progress, flagging uncertainty, deciding what to search next.
5. **Optional CoT+ReAct hybrid**: For knowledge-intensive tasks, the best results come from running ReAct first and falling back to CoT if the model signals low confidence.

Mathematically, if you think of the agent as producing a sequence of tokens, the action space is augmented: the model can emit either reasoning tokens (no environment effect) or action tokens (trigger an external call). The policy is:

$$\pi(a_t | c_t) \quad \text{where} \quad c_t = (o_1, a_1, \ldots, o_{t-1}, a_{t-1}, \text{thought}_t)$$

The context $c_t$ now includes both environment observations *and* the model's own prior thoughts — richer than pure RL baselines.

## Key Results

| Benchmark | Task Type | ReAct vs. Best Baseline | Notes |
|---|---|---|---|
| HotpotQA | Multi-hop QA | Competitive with CoT; CoT+ReAct is best overall | Wikipedia API access |
| Fever | Fact verification | Overcomes hallucination vs. CoT alone | Wikipedia API access |
| ALFWorld | Embodied text game | **+34% absolute success rate** vs. BUTLER (IL/RL) | 1-shot prompting |
| WebShop | Web navigation | **+10% absolute success rate** vs. IL/RL baseline | 2-shot prompting |

The ALFWorld and WebShop numbers are particularly striking: ReAct with a single in-context example beats methods trained on 10³–10⁵ task instances. That's the efficiency of good prompting with reasoning grounding.

## Limitations

- **Context length**: Long `Thought/Act/Obs` trajectories burn through context quickly. On tasks requiring many steps, the model can run out of room.
- **Prompt sensitivity**: Performance depends on the quality of the few-shot examples. Bad demonstrations propagate.
- **No learning**: Vanilla ReAct doesn't update weights — it can't systematically improve from past errors across episodes. The paper notes finetuning as future work.
- **Action space design**: The approach assumes someone has already defined sensible actions and built the tool connectors. It doesn't solve the "how do you even know what tools exist" problem.
- **Hallucination not eliminated**: ReAct reduces hallucination via grounding, but the model can still hallucinate *within* reasoning traces before retrieving the right observation.
- **Fixed LLM**: Evaluated on PaLM-540B and GPT-3. Smaller models may not generate coherent enough thoughts to benefit from the paradigm.

## Why It Matters

ReAct is one of the foundational papers for the modern **AI agent** paradigm. The `Thought/Action/Observation` loop it introduced is now the backbone of nearly every LLM agent framework — LangChain's agent executor, AutoGPT, OpenAI's function-calling patterns, and tool-use APIs all implement essentially this pattern.

The paper demonstrated that you don't need reinforcement learning or massive amounts of task-specific training data to get capable agent behavior — a few good demonstrations plus a capable base LLM and real tool access can dramatically outperform specialized systems. It also gave practitioners a *human-interpretable* agent: you can read the thought traces and understand exactly why the agent did what it did, which matters enormously for debugging and trust.

Conceptually, ReAct reframed the LLM-as-agent problem: instead of asking "how do we train an LLM to act?", it asked "can we prompt an LLM to reason its way through acting?" — and the answer was a clear yes.
