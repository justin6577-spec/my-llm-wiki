# S4: Structured State Space Sequence Model

> Albert Gu, Karan Goel, Christopher R├й, "Efficiently Modeling Long Sequences with Structured State Spaces", ICLR 2022 (arXiv:2111.00396)

## TL;DR

The idea behind S4 is to model sequences the way control theory would: as a continuous dynamical system

$$
x'(t) = Ax(t) + Bu(t), \quad y(t) = Cx(t) + Du(t)
$$

where $u$ is the input sequence, $x$ is a latent state, and $y$ is the output. Prior work (LSSL) showed that if you choose $A$ using the **HiPPO framework** (an optimal projection matrix derived to memorize input history), the model can theoretically capture dependencies at any range. But the LSSL required $O(N^2 L)$ computation тАФ completely impractical for any realistic model size or sequence length.

S4 fixes this by discovering a structure in $A$ that makes computation feasible: the HiPPO matrix can be written as **low-rank + normal**, which means it can be **diagonalized** (with a low-rank correction), reducing the SSM computation to a **Cauchy kernel** тАФ a well-studied operation that runs in $O(N + L)$ rather than $O(N^2 L)$.

The result: a sequence model that runs as a convolution at training time, as an $O(1)$-per-step recurrence at inference time, handles dependencies at 16,000+ steps, and outperforms all prior baselines on Long Range Arena тАФ including solving Path-X, a task where every prior model achieves тЙд50% (chance level).

## The Core Idea тАФ Three Views of the Same Model

A state space model has three equivalent computational representations. S4 switches between them depending on the setting:

### 1. Continuous-time ODE (for initialization)

$$
x'(t) = Ax(t) + Bu(t), \quad y(t) = Cx(t)
$$

This is where HiPPO lives: the matrix $A$ is derived to optimally project the input history onto a polynomial basis, giving the model a principled reason to remember things far in the past.

### 2. Discrete recurrence (for inference)

After discretizing with step size $\Delta$:

$$
\bar{A} = e^{\Delta A}, \quad \bar{B} = (A)^{-1}(e^{\Delta A} - I)B
$$

$$
x_k = \bar{A} x_{k-1} + \bar{B} u_k, \quad y_k = C x_k
$$

This runs in $O(N)$ per step тАФ constant time regardless of sequence length.

### 3. Global convolution (for training)

The recurrence unrolls into a convolution: $y = \bar{K} * u$ where the convolution kernel $\bar{K}$ is:

$$
\bar{K} = (C\bar{B},\; C\bar{A}\bar{B},\; C\bar{A}^2\bar{B},\; \ldots,\; C\bar{A}^{L-1}\bar{B})
$$

Computing this kernel is the hard part тАФ it requires powers of $\bar{A}$, which for the HiPPO matrix has $O(N^2)$ cost. S4's key contribution is that this reduces to a **Cauchy kernel**, computable in $O((N+L)\log^2(N+L))$ using fast algorithms, which is practically negligible compared to the convolution itself.

## Key Concepts

- **[[State Space Model]] (SSM)** тАФ the continuous-time ODE framework $x' = Ax + Bu$, $y = Cx + Du$; the backbone of all SSM-based sequence models including Mamba
- **[[HiPPO matrix]]** тАФ a specific $N \times N$ matrix $A$ derived to optimally memorize input history via polynomial projections; the theoretical foundation for long-range capability
- **[[HIPPO]]** тАФ "High-order Polynomial Projection Operators"; the mathematical framework behind the HiPPO matrix
- **[[Cauchy kernel]]** тАФ a structured matrix computation $K_{ij} = \frac{1}{\omega_i - \zeta_j}$ that arises from the diagonalized SSM; computable efficiently via divide-and-conquer
- **[[Low-rank correction]]** тАФ the trick that makes the HiPPO matrix diagonalizable: $A = \text{diagonal} + \text{rank-1 correction}$
- **[[Long Range Arena]]** (LRA) тАФ a standard benchmark of six tasks testing sequence models on dependencies up to 16K steps; S4 achieves state-of-the-art on all six, including Path-X which prior models cannot solve
- **[[Discretization]]** тАФ the process of converting the continuous-time ODE to a discrete recurrence via the ZOH (zero-order hold) rule
- **[[Convolutional view]]** тАФ unrolling the SSM recurrence into a global convolution; enables parallel training via FFT convolution

## Architecture / Method

S4 replaces the attention layer in a deep sequence model with an **S4 layer**:

```
Input u тИИ R^L
    тЖУ
S4 layer: u тЖТ conv(K╠Д, u)    [training: FFT convolution, O(L log L)]
          u тЖТ recurrence     [inference: O(1) per step]
    тЖУ
Gated activation (GELU)
    тЖУ
Output y тИИ R^L
```

Multiple S4 layers are stacked with skip connections (like a ResNet or Transformer). Each S4 layer has its own learned $A$, $B$, $C$, $\Delta$ тАФ but $A$ is initialized from the HiPPO matrix, giving it strong inductive bias for long-range memory from the start.

The parameters are:
- $A \in \mathbb{R}^{N \times N}$ тАФ the state transition (structured as low-rank + diagonal, ~$O(N)$ parameters)
- $B \in \mathbb{R}^{N \times 1}$, $C \in \mathbb{R}^{1 \times N}$ тАФ input and output projections
- $\Delta \in \mathbb{R}$ тАФ timescale (discretization step)

## Comparison to Prior Work

- vs. **[[Transformer]]** тАФ same quality on LM tasks, but $O(1)$ inference per step vs. $O(T^2)$ attention; 60├Ч faster generation at sequence length 16K.
- vs. **[[LSTM]] / GRU** тАФ also $O(1)$ per step, but LSTM can't capture very long-range dependencies without vanishing gradients; S4 has principled long-range memory via HiPPO.
- vs. **Linear attention (Performer, Linformer)** тАФ those are approximations with accuracy loss; S4 is a different architecture altogether, not an attention approximation.
- vs. **[[Mamba]]** тАФ Mamba (2024) extends S4 with **input-selective** state transitions ($A$, $B$, $C$ depend on the input token), fixing S4's main weakness. S4 uses time-invariant (fixed) dynamics; Mamba makes them content-aware.

---

## Limitations

- **Time-invariant dynamics.** S4's $A$, $B$, $C$ are fixed for the entire sequence тАФ the model can't selectively attend to specific tokens based on content. This limits performance on recall-heavy tasks. [[Mamba]] fixes this with selective SSMs.
- **Initialization sensitivity.** The HiPPO initialization is critical; poorly initialized S4 layers converge slowly or not at all.
- **Less understood at scale.** S4 was benchmarked up to medium scale; the scaling laws at GPT-scale are better established for Mamba (S4's successor).
- **Complex implementation.** The Cauchy kernel computation and the numerical handling of the non-normal matrix require careful implementation.

---

## Why It Matters

- **S4 is the direct predecessor of Mamba.** The entire SSM family (S4 тЖТ S5 тЖТ Mamba тЖТ Mamba-2) traces to this paper. Understanding S4 is understanding where selective state spaces came from.
- **It proved SSMs can match Transformers on long sequences.** Before S4, the consensus was that attention was necessary for tasks requiring long-range dependencies. S4 showed a structured recurrence is sufficient.
- **It introduced the HiPPO-inspired parameterization** that every SSM after it inherits: structured $A$, discretization step $\Delta$, and the convolution-recurrence duality.
- **Path-X was a watershed moment.** Solving a 16K-step task that all prior architectures fail is a concrete proof that SSMs can handle what attention approaches (let alone RNNs) cannot.
